### 구현 기능 목록

## 금액 입력 받기
- 1000원 단위로 입력을 받는다.(-> 그렇지 않은 경우 예외처리)
- 예외처리
    1. 1000원 단위가 아닌 경우
    2. 숫자가 아닌 경우
    3. 0 이하인 경우

## 구입한 로또의 개수 구하기
- 구입 금액을 1000으로 나누어 몇개의 로또를 구입했는지 구한다.

## 로또 숫자 발행하기
- 구입한 로또의 개수만큼 로또를 발행한다.
- 1 ~ 45의 숫자를 중복되지 않게 발행한다.
    (MissionUtils 라이브러리 사용)

## 당첨 번호 입력 받기
- 번호는 쉼표(,)를 기준으로 구분한다.
- 1 ~ 45 의 숫자를 중복되지 않게 입력받는다.
- 예외처리
    1. 숫자가 아닌 경우
    2. 1 ~ 45 사이의 숫자가 아닌 경우
    3. 6개가 아닌 경우


## 보너스 번호 입력 받기
- 1 ~ 45 사이의 숫자 하나를 입력 받는다
- 예외처리
    1. 숫자가 아니 경우
    2. 1 ~ 45 사이의 숫자가 아닌 경우
    3. 당첨 번호와 중복되는 숫자를 입력한 경우

## 당첨 여부 검사하기
- 로또 숫자와 당첨 번호 비교하여 당첨 여부 검사
    - 1등: 6개 번호 일치 / 2,000,000,000원
    - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
    - 3등: 5개 번호 일치 / 1,500,000원
    - 4등: 4개 번호 일치 / 50,000원
    - 5등: 3개 번호 일치 / 5,000원
- 당첨 내역 출력

## 수익률 계산
- 수익률 = (당첨된 금액)/(구입 금액) * 100(%)
- 소수점 둘째 자리 반올림
- 수익률 출력

### 추가된 요구 사항
- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현한다.
  - 함수(또는 메서드)가 한 가지 일만 잘 하도록 구현한다.
- else를 지양한다.
  - 힌트: if 조건절에서 값을 return하는 방식으로 구현하면 else를 사용하지 않아도 된다.
  - 때로는 if/else, switch문을 사용하는 것이 더 깔끔해 보일 수 있다. 어느 경우에 쓰는 것이 적절할지 스스로 고민해 본다.
- 도메인 로직에 단위 테스트를 구현해야 한다. 단, UI(Console.readLine, Console.print) 로직에 대한 단위 테스트는 제외한다.
  - 핵심 로직을 구현하는 코드와 UI를 담당하는 로직을 구분한다.
  - 단위 테스트 작성이 익숙하지 않다면 `__tests__/LottoTest.js`를 참고하여 학습한 후 테스트를 구현한다.
